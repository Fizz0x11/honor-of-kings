# GitHub & Gitee 仓库管理指南

> 本文档记录如何批量设置仓库私有权限，以及如何将 OpenCode Skill 推送到开源仓库。

---

## 一、GitHub 批量设置私有

### 1.1 获取 GitHub Token

```bash
# 方式1: 使用 gh CLI
gh auth token

# 方式2: 直接从 ~/.netrc 读取
grep "github.com" ~/.netrc | awk '{print $4}'
```

### 1.2 批量设置私有

```bash
# 设置单个仓库为私有
curl -s -X PATCH "https://api.github.com/repos/{username}/{repo-name}" \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github+json" \
  -H "Content-Type: application/vnd.github+json" \
  -d '{"private":true}'

# 示例：设置 hermes-memory 为私有
curl -s -X PATCH "https://api.github.com/repos/Fizz0x11/hermes-memory" \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github+json" \
  -H "Content-Type: application/vnd.github+json" \
  -d '{"private":true}'
```

### 1.3 删除仓库

```bash
# 删除仓库（需谨慎！）
curl -s -X DELETE "https://api.github.com/repos/{username}/{repo-name}" \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github+json"
```

### 1.4 批量脚本示例

```bash
#!/bin/bash
# set-private.sh - 批量设置仓库为私有

GITHUB_TOKEN="your_token_here"
USERNAME="your_username"

repos=(
  "hermes-memory"
  "opencode-md"
  "cc-md"
  "Claw-bot"
  "opencode-skills"
)

for repo in "${repos[@]}"; do
  echo "Setting $repo to private..."
  curl -s -X PATCH "https://api.github.com/repos/$USERNAME/$repo" \
    -H "Authorization: Bearer $GITHUB_TOKEN" \
    -H "Accept: application/vnd.github+json" \
    -H "Content-Type: application/vnd.github+json" \
    -d '{"private":true}'
  echo " Done: $repo"
done
```

## 二、Gitee 批量设置私有

### 2.1 获取 Gitee Token

在 Gitee 设置 → 私人令牌 → 生成新令牌（需要 repo 权限）

### 2.2 设置仓库为私有

```bash
# Gitee API 格式
curl -s -X PATCH "https://gitee.com/api/v5/repos/{username}/{repo-name}" \
  -H "Content-Type: application/json" \
  -d '{"access_token":"your_token","private":true,"name":"repo-name"}'

# 注意：Gitee API 要求 name 字段不能为空
```

### 2.3 批量脚本示例

```bash
#!/bin/bash
# gitee-set-private.sh - 批量设置 Gitee 仓库为私有

GITEE_TOKEN="your_token_here"
USERNAME="your_username"

repos=(
  "nsd2019"
  "warcraft-coach"
)

for repo in "${repos[@]}"; do
  echo "Setting $repo to private..."
  curl -s -X PATCH "https://gitee.com/api/v5/repos/$USERNAME/$repo" \
    -H "Content-Type: application/json" \
    -d "{\"access_token\":\"$GITEE_TOKEN\",\"private\":true,\"name\":\"$repo\"}"
  echo " Done: $repo"
done
```

## 三、推送 Skill 到开源仓库

### 3.1 创建 GitHub 仓库

```bash
# 方式1: 使用 GitHub CLI
gh auth login
gh repo create honor-of-kings --public --clone

# 方式2: 使用 API
curl -s -X POST "https://api.github.com/user/repos" \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github+json" \
  -H "Content-Type: application/vnd.github+json" \
  -d '{"name":"honor-of-kings","description":"Honor of Kings professional coach skill","public":true}'
```

### 3.2 创建 Gitee 仓库

```bash
curl -s -X POST "https://gitee.com/api/v5/user/repos" \
  -H "Content-Type: application/json" \
  -d '{"access_token":"your_token","name":"honor-of-kings","description":"Honor of Kings professional coach skill","public":true}'
```

### 3.3 初始化本地仓库并推送

```bash
cd D:/FizzDesktop/Download/opencode-md/oc-skills/honor-of-kings

# 添加远程仓库
git remote add origin "https://ghp_YOUR_TOKEN@github.com/username/honor-of-kings.git"
git remote add gitee "https://username:YOUR_TOKEN@gitee.com/username/honor-of-kings.git"

# 推送前先创建 .gitignore 排除敏感文件
echo "oldbak/" > .gitignore
echo ".DS_Store" >> .gitignore

# 提交并推送
git add -A
git commit -m "feat: initial release of honor-of-kings skill"
git push origin master
git push gitee master
```

### 3.4 完整推送脚本

```bash
#!/bin/bash
# push-skill.sh - 推送 Skill 到 GitHub 和 Gitee

SKILL_NAME="honor-of-kings"
GITHUB_TOKEN="ghp_xxxx"
GITEE_TOKEN="xxxx"
USERNAME="your_username"
EMAIL="your@email.com"

SKILL_PATH="D:/FizzDesktop/Download/opencode-md/oc-skills/$SKILL_NAME"

cd "$SKILL_PATH" || exit

# 初始化（如果是新仓库）
if [ ! -d ".git" ]; then
  git init
  git config user.name "$USERNAME"
  git config user.email "$EMAIL"

  # 添加远程
  git remote add origin "https://$GITHUB_TOKEN@github.com/$USERNAME/$SKILL_NAME.git"
  git remote add gitee "https://$USERNAME:$GITEE_TOKEN@gitee.com/$USERNAME/$SKILL_NAME.git"
fi

# 创建 .gitignore
if [ ! -f ".gitignore" ]; then
  cat > .gitignore << 'EOF'
oldbak/
.DS_Store
*.log
EOF
fi

# 提交并推送
git add -A
git commit -m "feat: release $SKILL_NAME skill"
git push origin master
git push gitee master

echo "Push complete!"
```

## 四、常见问题

### 4.1 GitHub Fork 无法设为私有
```
"Public forks can't be made private"
```
**解决方案**：需要联系原仓库作者，将 Fork 转为独立仓库，或手动在 GitHub 网页设置。

### 4.2 Gitee API 返回 "name is missing"
```bash
# 错误
-d '{"access_token":"token","private":true}'

# 正确（name 不能为空）
-d '{"access_token":"token","private":true,"name":"repo-name"}'
```

### 4.3 Git 推送 remote 不存在
```bash
# 检查远程仓库配置
git remote -v

# 添加缺失的远程
git remote add origin "https://..."
git remote add gitee "https://..."
```

### 4.4 GitHub Token 类型

| Token 类型 | 权限范围 |
|-----------|----------|
| `ghp_xxx` | Personal Access Token (PAT) |
| `gho_xxx` | OAuth App |
| `ghu_xxx` | GitHub App |

建议使用 PAT（`ghp_`），权限更清晰。

## 五、仓库权限检查清单

发布前检查清单：

- [ ] 仓库名称正确（使用 `honor-of-kings` 而非 `warcraft-coach`）
- [ ] README 中的目录结构名称已更新
- [ ] `.gitignore` 已排除 `oldbak/` 等敏感目录
- [ ] 无硬编码凭证或密钥
- [ ] 仓库设为 public（开源）或 private（私有）
- [ ] GitHub 和 Gitee 仓库都已推送
